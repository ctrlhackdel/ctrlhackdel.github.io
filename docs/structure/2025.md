# 2025 Project Structure
For 2025, we decided to try a single repository to handle everything from the dashboard to the landing page. While dev in theory was simpler, the organization was messy and components overlapping made it more difficult to manage, including assets.
```
ctrl-hack-del-2025/
├── src/
│   ├── app/                   # Next.js App Router pages
│   │   ├── (auth)/            # Authentication pages (sign-in, sign-up)
│   │   ├── (dashboard)/       # Protected dashboard pages
│   │   ├── (landing)/         # Public landing pages
│   │   └── api/               # All API routes
│   ├── components/            # Reusable UI components
│   │   ├── auth/              # Authentication-related components
│   │   ├── layout/            # Layout components (navbar, footer)
│   │   ├── ui/                # Base UI components (buttons, inputs)
│   │   └── emails/            # Email templates
│   ├── lib/                   # Client-safe utilities and configurations
│   ├── server/                # Server-only code
│   │   ├── db/                # Database configuration and schema
│   │   └── server-auth.ts     # Server-side auth utilities
|   |   └── user.ts            # User-related helper functions such as getUserById, updateUserName, etc
|   |   └── [anything-else].ts # Add more files as needed as such applications.ts
│   ├── config/                # Application configuration
│   └── types/                 # TypeScript type definitions
├── public/                    # Static assets
└── package.json
```
