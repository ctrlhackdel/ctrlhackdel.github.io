# 2024 Project Structure
Development was made to split into multiple teams, focusing on different components. Having the separate repos helped to keep track of everything individually, and in theory made it better to organize.
## Landing
```
2024.ctrlhackdel.com_v0.x/
├── app/                    # Next.js App Router pages
│   ├── (landing)/          # Public landing (Possibly proposed to have a single repo)
│   │   ├── _components/    # Reusable components for each section of the landing page
│   ├── api/                # API routes for pre-register emails and unsubscribe links
│   ├── privacy/            # Privacy page
│   ├── projects/           # Projects page
│   │   ├── _components/    # For some reason custom components were used?
│   └── unsubscribe/        # Unsubscribe page
├── components/             # Components that are shared across all the different landing sections
├── data/                   # Used for pre-registration interest
├── lib/                    # Client safe utility and configuration
│   ├── constants/          # Fixed variables
│   ├── db/                 # Pre-registration database
│   ├── emails/             # Confirm pre-registration email
│   ├── fonts/              # Custom font
│   └── validators/         # Zod Schemas
├── public/                 # All static assets
├── scripts/                # Used to initialize and get the pre-registration list
└── package.json
```
## Dashboard
```
dashboard-chd-2024/
├── actions/                    # Scripts used for server-specific calls, including db pulls and S3
├── app/                        # Next.js App Router pages
│   ├── (auth)/                 # Authentication Pages (login, register, oauth)
│   ├── (main)/                 # Main Dashboard Pages
│   ├── api/                    # API Routes for client initialized calls (i.e. save application)
├── auth.ts                     # Authentication handler (handled by Lucia -- Deprecated)
├── components/                 # All the components used within the dashboard, including navbar and input
│   ├── hackerdex/              # Components specific to the hackerdex
│   └── ui/                     # UI and ShadCN specific components
├── data/                       # MySQL Database calls and S3 handling on client side 
├── lib/                        # Client safe utilities and configurations
│   ├── config/                 # SEO and HTML Header
│   ├── constants/              # Constants used throughout the site
│   ├── db/                     # Database schema and AWS RDS configuration
│   ├── fonts/                  # Font configuration for custom fonts
│   ├── hooks/                  # Hooks that needed to be used universally
│   ├── ses/                    # All functions for Simple Email Service
│   ├── utils/                  # Used to preconfigure layout for 2024 schedule (it was cool)
│   └── validators/             # Zod schemas
├── public/                     # All public assets
│   ├── apple/                  # Apple icon for mobile
│   ├── assets/                 # All images used throughout the site
│   ├── avatars/                # Clothing used for the hackathon avatars
├── types/                      # Definition of questions, schemas, and any interfaces/enumerations needed
└── us-east-2-bundle.pem
```
